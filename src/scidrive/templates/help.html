SciDrive is a network storage for scientific data. We provide free storage to scientists who want to publish their data without need to organize it in any way or have own infrastructure to support it.

To start using SciDrive, create a folder using the "Create folder" button. Double click a folder to view its contents.

Once you are inside a folder you can start uploading files by Drag-and-Dropping these to the browser window.

To upload, drag and drop one or multiple files from your local drive into your browser window.
(Folder uploading is not supported by browsers yet)
To pull any http URL contents into existing file, choose "Pull from URL..." from the file's context menu:

The data will be asynchronously downloaded by the service.

You can only upload files into a folder. Uploading files to root is not permitted.

Files and folders have right-click context menu for operations on them.

To select one or multiple files, left-click one file or drag over multiple files. These selected items can be drag-and-dropped between panels to organize the space and transfer files between remote locations.

The panel is usually updated after user's operations. To manually refresh the contents, use Refresh toolbar button.

Files can be downloaded by double-clicking on it. Also to download a file or folder with all its contents, right-click on it and choose Download from context menu.

Folders are downloaded as .tar archives.
Online Preview is available for files recognized as scidrive/resources.

The right dropdown element contains the list of available regions. Currently new SSO users registration is only supported in JHU region. SDSC only works for current users. Please send request if you need to be added to San Diego Supercomputing Center mirror users.

The star before the region ID means the user is authenticated in this region using Single Sign On (SSO) service. When choosing the region with no star, the user will be asked to authenticate. The Log off removes the stored user credentials for currently selected region and opens the first authenticated one. If user is not currently authenticated in any region, he will be redirected to default region SSO authentication page.

To get world-accessible link to a file or folder, choose "Quick Share URL..." from file or folder context menu

Containers will be downloaded as .tar archives.

You can share a top-level container for read-only or read+write access by a group of users or by everyone. To access the share dialog, choose "Share" menu item from the context menu of top-level container.

The dialog allows choosing the predefined group for the share or make it public.

"Create the share" button opens the dialog with new share URL and ID. The URL provides Web view of the share.

The share ID can be used to provide access to the share for console scripts. To get the OAuth token for shared container, the "share" parameter should be added to request_token and authorize OAuth URLs:

http://server/vospace-2.0/request_token?share=<share_id>

http://server/vospace-2.0/authorize?provider=vao&share=<share_id>

When accessing the authorize URL with anonymous share ID, no user acton is needed. The URL can be queried by the script in automatic mode, as described here: https://github.com/dimm0/scidrive/wiki/Using-Dropbox-SDK

The service provides automatic extraction of metadata from uploaded files in the background. Filetypes are automatically recognized by its contents signatures. For some filetypes there are plugins to extract metadata and store it in side services. The needed plugins should be enabled using the metadata exctractors configurator.

The available metadata extractors arelisted on the left of the dialog. The symbol in extrctor name shows the current enabled/disabled status of the plugin. To change the status, use the switch and save button to save the state to the service.

The service supports both VOSpace 2.0 and Dropbox REST API to access the data.

The specification can be found at http://www.ivoa.net/Documents/VOSpace. The service supports both X.509 certificate and OAuth+OpenID as Single Sign On preferred way of authentication.
The specification can be found at Dropbox API. The service supports OAuth+OpenID as Single Sign On preferred way of authentication.
